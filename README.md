# Система автоматизированного бенчмаркинга вычислительных алгоритмов

Сервис для автоматизированного сравнения производительности вычислительных алгоритмов на разных языках программирования. Проект объединяет веб-платформу, PostgreSQL-хранилище результатов и десктопную утилиту, которая запускает тесты локально на машине пользователя.

## О проекте

Проект предназначен для автоматизированного тестирования алгоритмов математической физики и численных вычислений: операций с матрицами, построения фракталов, случайных блужданий, клеточных автоматов и моделей распространения эпидемий.

Поддерживаемые языки:

- **Python**
- **C++**
- **CUDA**
- **Go**
- **MATLAB**

Ключевые возможности:

- **Сбор аппаратных метрик:** CPU, GPU и общий объем RAM фиксируются вместе с результатом запуска.
- **Изолированный замер времени:** эталонные скрипты измеряют только вычислительную часть алгоритма.
- **Интерактивная визуализация:** графики и аналитика строятся во frontend с использованием Chart.js.
- **Защита от спама:** регистрация ограничена in-memory rate limiting.
- **Модерация пользовательского кода:** пользовательские решения отправляются на проверку администратору.
- **Глобальный лидерборд:** лучшие результаты отображаются в рейтинге по задачам и языкам.
- **Паспорт задачи:** отдельная динамическая страница с теорией, техническим заданием и эталонными реализациями.

## Архитектура

**Backend**

- FastAPI

- HTML5

**Client**

- Docker

- Production-конфигурация через `docker-compose.prod.yml`

## Развертывание сервера

1. Склонируйте репозиторий:

```bash
git clone https://github.com/grigory-ag/bench2257
cd bench2257
```

2. Создайте файл окружения на основе шаблона:

```bash
cp .env.example .env
```

3. Заполните значения в `.env`:

```env
POSTGRES_USER=bench_user
POSTGRES_PASSWORD=change_me
POSTGRES_DB=bench_db
DATABASE_URL=postgresql://bench_user:change_me@db:5432/bench_db
SECRET_KEY=change_me_to_a_long_random_secret
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
```

4. Запустите production-сборку:

```bash
docker-compose -f docker-compose.prod.yml up -d --build
```

5. Откройте веб-интерфейс:

```text
http://localhost/
```

## Development & Client

### Сборка клиентского приложения

Десктопная утилита находится в `benchmark_client/app.py`. Для сборки `.exe` используется `build_client.py`.

Установите зависимости клиента и PyInstaller, затем выполните:

```bash
python build_client.py
```

Скрипт сборки требует установленный `pyinstaller`. После сборки бинарный файл клиента должен быть доступен как `benchmark_client/BenchmarkClient.exe`, чтобы backend мог включить его в скачиваемый ZIP-архив.

### Генерация тестовых данных

Матрицы для задач с линейной алгеброй генерируются скриптом:

```bash
python benchmark_client/scripts/data_generator.py --n 4000 --out-dir benchmark_client
```

Файлы `M1.dat`, `M2.dat` и `T.dat` не хранятся в репозитории и не включаются в скачиваемый архив. Десктопная утилита умеет генерировать их автоматически перед первым запуском теста.

### Локальный запуск клиента из исходников

```bash
cd benchmark_client
python app.py
```

Для запуска эталонов на разных языках должны быть установлены соответствующие инструменты: Python, `g++`, CUDA Toolkit с `nvcc`, Go и MATLAB.

## Единый стандарт алгоритмов

Чтобы результаты были сопоставимыми, все эталонные и пользовательские алгоритмы должны соблюдать единый формат.

Правила:

- **I/O операции не входят в измеряемый участок.** Чтение входных данных, запись файлов и подготовка окружения выполняются до старта таймера или после его остановки.
- **В stdout выводится ровно одна строка JSON:**

```json
{"execution_time_ms": 123.45}
```

- **Интерактивный ввод запрещен.** Скрипт не должен ожидать действия пользователя, чтения из консоли или подтверждения.
- **Визуализация отключена.** Нельзя вызывать `plot`, `imshow`, `figure`, `pause` и аналогичные функции в эталонном режиме.
- **Лишний stdout запрещен.** Отладочные сообщения, прогресс-бары и текстовые пояснения мешают парсеру результата.
- **Тяжелые вычисления должны быть устойчивы к OOM.** Для крупных массивов рекомендуется батчевая обработка и экономные типы данных.

Минимальный пример JSON-вывода на Python:

```python
import json
import time

start = time.perf_counter()
# math-only benchmark section
elapsed_ms = (time.perf_counter() - start) * 1000

print(json.dumps({"execution_time_ms": elapsed_ms}, separators=(",", ":")))
```

Минимальный пример JSON-вывода на MATLAB:

```matlab
tStart = tic;
% math-only benchmark section
elapsed = toc(tStart);
ms = elapsed * 1000;
fprintf('{"execution_time_ms": %.2f}\n', ms);
```
